# Glossary

- `SUT`: System Under Test. En este repo es `lightning-player`.
- `Observability`: señales externas confiables para validar comportamiento.
- `Primary signal`: señal más confiable para una aserción.
- `Secondary signal`: señal útil como apoyo, no como fuente principal.
- `False positive`: test que pasa o falla por una señal incorrecta, timing o supuesto inválido.
- `Test brief`: contrato mínimo que debe existir antes de generar un test con IA.
- `Feature spec`: descripción funcional de una feature.
- `Business rule`: restricción o comportamiento esperado del producto.
- `Deterministic test`: test que no depende de condiciones externas inestables.
