# Taxonomy

## Tipos de documento

- `sut-overview`: contexto global del sistema bajo prueba
- `feature-spec`: definición funcional de una feature
- `business-rules`: reglas de negocio explícitas
- `observability`: señales confiables e inestables
- `edge-cases`: casos borde y restricciones
- `test-strategy`: plan de pruebas por tipo
- `coverage-report`: estado de cobertura
- `risk-map`: mapa de riesgo
- `pipeline-contract`: reglas del pipeline de IA
- `template`: plantilla reusable
- `runbook`: procedimiento operativo
- `decision`: decisión técnica
- `session-note`: aprendizaje de una sesión

## Relación esperada

`feature-spec -> business-rules -> observability -> edge-cases -> test-strategy -> test-brief -> spec de tests`
